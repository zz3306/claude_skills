AI 到底是什么 What is AI, Really?

用了一年 AI 之后, 你说得清"AI"、"机器学习"、"深度学习"、"Transformer"、"LLM"到底什么关系吗? 这章把概念套娃理清楚, 历史用一段话说完, 然后给 Ch2 的深入做好准备。 After a year of using AI — can you actually explain how "AI", "ML", "DL", "Transformer", "LLM" relate? This chapter clears the nesting up, covers history in one paragraph, and sets up Ch2's deep dive.

⏱️ 15 min 📖 概念入门, Ch2 深入Concept primer; Ch2 goes deep

01 · 电梯定义01 · The Elevator Definition

AI (Artificial Intelligence) 一句话: 让机器表现出"看起来很智能"的行为的所有技术总称。 AI in one sentence: the umbrella term for any technology that makes a machine behave in ways we'd call "intelligent".

注意 "看起来" 这三个字。 AI 不是"机器有了意识", 而是"机器能完成原本需要人智能才能完成的任务"。 这两件事不是一回事。 Notice "behave like". AI is NOT "machines become conscious" — it's "machines do things that used to require human intelligence". Two very different claims.

深入: AI 的三种定义路线Deeper: three schools of defining AI +3 min

在 AI 学界历史上, 至少有三种主流定义: Historically, the AI field has had at least three definitional schools:

  1. 行为派 (Turing 学派):Behavioral (Turing school): 看起来像人就算。 1950 年图灵测试就是这一派 —— 你跟它聊天分不清是人是机器, 它就算 AI。 现代 ChatGPT 是这派的胜利。 If it acts like a human, it's AI. Turing Test (1950) is this school's flagship — if you can't tell it from a human in chat, it's AI. ChatGPT is this school's victory.
  2. 理性派 (符号派):Rationalist (symbolic): 能像人一样推理就算。 1950-80 年代主流, 试图把"知识"用规则和符号写出来 (专家系统)。 后来发现"我们自己都说不清自己是怎么思考的", 走入死胡同。 If it can reason like a human, it's AI. Dominant from 1950s-80s. Tried to encode knowledge as rules (expert systems). Hit a wall: we ourselves can't articulate how we think.
  3. 数据派 (统计学派, 现在的主流):Data-driven (statistical, today's mainstream): 不要试图教机器"怎么思考", 给它海量数据让它自己找规律。 这就是 Machine Learning 的核心思想, 也是今天所有 AI 的底层范式。 Don't teach machines "how to think" — feed them massive data, let them find patterns. This is Machine Learning's core idea and underpins all modern AI.

所以今天你用的 ChatGPT, 是"数据派"的产物, 但它表现出的能力让"行为派"的目标 (图灵测试) 在 2023 年被认为基本完成。 So today's ChatGPT is a data-driven creation, yet its capabilities have basically completed the behaviorist goal (Turing test, around 2023).

02 · 概念地图: 5 个词到底什么关系02 · Concept Map: How the 5 Terms Relate

这是大多数人最混乱的地方。 来, 我们把它一次说清楚: This is where most people get muddled. Let's clear it up once and for all:

AI (人工智能 — 最大的圈Artificial Intelligence — biggest circle)
  ⊃ Machine Learning (机器学习 — AI 的一个流派a school within AI)
    ⊃ Deep Learning (深度学习 — ML 的一种方法a method within ML)
      ⊃ Transformer (深度学习的一种结构an architecture within DL)
        ⊃ LLM (用 Transformer 训练的大语言模型large language model using Transformer)
          ⊃ GenAI (生成式 AI — LLM 的用法之一one use case of LLMs)

点开下面每个概念看详细解释:Click each below for the full explanation:

🟢 AI (Artificial Intelligence) — 人工智能Artificial Intelligence

一句话: 所有让机器"做智能事"的方法 —— 包括但不限于机器学习。 One line: any technique that makes a machine do intelligent things — including but not limited to machine learning.

非机器学习的 AI 例子:Non-ML examples of AI:

  • 下棋程序 (深蓝, 1997 战胜卡斯帕罗夫的那个) —— 靠暴力搜索 + 人写的评估函数, 没机器学习 Chess programs like Deep Blue (1997, beat Kasparov) — brute-force search + handcrafted evaluation, no ML
  • 早期搜索引擎 —— 关键词匹配 + PageRank, 没机器学习 Early search engines — keyword matching + PageRank, no ML
  • 导航 GPS 找最短路径 —— Dijkstra 算法, 没机器学习 GPS shortest path — Dijkstra's algorithm, no ML

所以"AI"是一个很老的词 (1956 年达特茅斯会议提出), 它包含了今天的机器学习, 也包含 60 年前那种"用规则硬编"的早期 AI。 "AI" is an old term (coined at Dartmouth in 1956). It encompasses today's ML and also the rule-based "good old-fashioned AI" of the 60s.

🟢 Machine Learning (ML) — 机器学习

一句话: 不要明确告诉机器规则, 给它数据让它自己找规律。 One line: don't write down the rules — give the machine data, let it find the pattern itself.

核心区别:The fundamental shift:

传统编程Traditional code 机器学习Machine Learning
你给You give 规则 + 数据Rules + Data 数据 + 答案Data + Answers
机器给Machine gives 答案Answers 规则Rules (the "model")
例子Example "如果邮件里有'中奖', 标记为垃圾""If email has 'winner', mark as spam" 看 100 万封人工标好的邮件, 自己找出"什么样的邮件像垃圾"Show it 1M labeled emails, it learns "what spam looks like"

3 个主要类型 (一句话各):3 main types (one line each):

  • Supervised Learning (监督学习): 数据带标签 (邮件 + "是不是垃圾") data comes with labels (email + "is it spam")
  • Unsupervised Learning (无监督学习): 数据没标签, 让机器自己找聚类 (比如用户分群) no labels — machine finds clusters (user segmentation)
  • Reinforcement Learning (强化学习): 通过"奖励/惩罚"学习 (AlphaGo、自动驾驶) learn via reward/penalty (AlphaGo, self-driving)
🟢 Deep Learning — 深度学习

一句话: 用"多层神经网络"来做 ML —— "深度"指的就是层数多。 One line: ML done with "many-layered neural networks" — "deep" means "many layers".

为什么能赢: 传统 ML 需要人类先告诉机器"哪些特征重要" (比如做垃圾邮件识别, 你得先列出"是否含'中奖'、是否含全大写..."这些特征)。 深度学习不需要 —— 你直接喂原始数据, 网络自己从底层逐层提取特征。 Why it won: classical ML needs humans to define features ("has 'winner', is all caps...") first. Deep learning doesn't — you feed raw data, and each layer extracts features automatically.

类比: 学认猫Analogy: learning to recognize a cat

  • 第 1 层学到"边缘和角"Layer 1 learns edges and corners
  • 第 2 层学到"由边缘组合成的耳朵、眼睛形状"Layer 2 learns ear-shapes, eye-shapes built from edges
  • 第 3 层学到"耳朵 + 眼睛 + 胡须 = 猫脸"Layer 3 learns "ears + eyes + whiskers = cat face"
  • 第 4 层学到"猫的全身姿态"Layer 4 learns full-body cat poses

这种"由简到繁、自动学层级特征"的能力, 是深度学习区别于其他 ML 的核心。 This automatic hierarchical feature learning is what sets deep learning apart from other ML.

关键时刻: 2012 年 AlexNet 在 ImageNet 图像识别比赛上断崖式碾压传统方法, 深度学习从此爆发。 Watershed moment: AlexNet at ImageNet 2012 obliterated classical methods. Deep learning has dominated ever since.

🟢 LLM (Large Language Model) — 大语言模型

一句话: 用 Transformer 结构 + 海量文本训练出的, 能"理解和生成自然语言"的大型神经网络。 One line: a large neural network trained on massive text using the Transformer architecture, capable of understanding and generating natural language.

"大"在哪里:What's "large":

  • 参数大: 几百亿到上万亿。 参数就是神经网络里的可调旋钮。 GPT-3 有 1750 亿参数, Claude Opus 4 据估计 1 万亿+。 Parameters: tens to thousands of billions. Parameters are the tunable knobs in the network. GPT-3 had 175B. Claude Opus 4 is estimated at 1T+.
  • 数据大: 训练时见过几乎整个互联网的文字 —— Wikipedia、新闻、书籍、论坛、代码... Data: trained on nearly all the text on the internet — Wikipedia, news, books, forums, code...
  • 算力大: 训练一次要几千张顶级 GPU 跑几个月, 成本上亿美元。 Compute: one training run uses thousands of top GPUs for months, costing hundreds of millions of dollars.

下一章 (Ch2) 我们专门讲 LLM 是怎么工作的。 Next chapter (Ch2) goes deep on how LLMs actually work.

🟢 GenAI (Generative AI) — 生成式 AI

一句话: 不只是"判断", 还能"生成"新内容的 AI —— 文字、图像、音频、视频、代码。 One line: AI that doesn't just classify or predict, but creates new content — text, images, audio, video, code.

对比:Compare:

  • 判别式 AI (Discriminative): 给我一张图, 我告诉你是不是猫 Discriminative AI: give me an image, I tell you if it's a cat
  • 生成式 AI (Generative): 我画一张猫给你 Generative AI: I'll paint you a cat

ChatGPT、Claude、Midjourney、Sora 都是 GenAI。 GenAI 这个词在 2022 年 ChatGPT 出来后才彻底火起来 —— 它不是新技术, 是 LLM 应用的新名字。 ChatGPT, Claude, Midjourney, Sora — all GenAI. The term exploded after ChatGPT (Nov 2022). Not new tech, just a new label for LLM applications.

03 · 一段话讲完 AI 70 年03 · 70 Years of AI in One Paragraph

AI 作为学科诞生于 1956 年达特茅斯会议, 经历过两次"寒冬" (60-70 年代, 80-90 年代承诺没兑现 + 资助断绝), 1997 年深蓝下棋赢卡斯帕罗夫, 2012 年 AlexNet 在 ImageNet 比赛碾压传统方法, 深度学习春天开始。 2017 年 Google 发 "Attention is All You Need"——Transformer 架构诞生, 这是真正的分水岭。 2022 年 11 月 ChatGPT 上线, 5 天 100 万用户, AI 从学术圈进入日常生活。 2024-2025 进入"agent 元年"和"推理模型"范式 (o1 / R1 / Extended Thinking, Ch3 §02c 会讲)——现在你正活在 AI 历史上最快变化的两年。 AI as a field was born at the 1956 Dartmouth Workshop, survived two "winters" (60-70s and 80-90s — broken promises + dried funding), saw Deep Blue beat Kasparov in 1997, then in 2012 AlexNet obliterated traditional methods at ImageNet — the deep learning spring began. In 2017 Google published "Attention is All You Need" — the Transformer architecture is born, the real dividing line. In Nov 2022 ChatGPT launched (1M users in 5 days), AI moved from academia to daily life. 2024-2025 brought the "year of agents" and "reasoning model" paradigm (o1 / R1 / Extended Thinking, see Ch3 §02c) — right now you're living through the fastest two years in AI history.

关键认知: 你只需要记 2017 (Transformer 论文) 和 2022.11 (ChatGPT) 这两个日期。 前者是技术分水岭, 后者是社会分水岭。 其它年份不重要。 Key insight: only two dates matter — 2017 (Transformer paper) and Nov 2022 (ChatGPT). The first is the technical watershed, the second is the social one. Other years aren't important.

04 · AI 不是什么 — 破除 5 个迷思04 · What AI is NOT — Debunking 5 Myths

❌ 迷思 1: "AI 像 Google, 是搜索引擎" Myth 1: "AI is like Google — a search engine"

事实: AI 不查任何东西, 它在"按概率续写"。 你问它"鲁迅是谁", 它不是去查百科, 而是基于训练时见过的所有提到鲁迅的文字, 生成一段最可能的回答。 Reality: AI doesn't look anything up. It probabilistically continues text. When you ask "who is Lu Xun?", it's not querying an encyclopedia — it's generating the most likely response based on all the Lu Xun text it saw in training.

后果: 它训练时没见过的事 (比如本周的新闻), 它要么不知道, 要么会编一个看起来很合理的答案 (幻觉)。 Consequence: things it didn't see in training (this week's news), it either won't know, or it'll fabricate a plausible-looking answer (hallucination).

❌ 迷思 2: "AI 有了意识/感情" Myth 2: "AI has consciousness / feelings"

事实: AI 输出"我感到很沮丧"不代表它真的沮丧。 那是它学到的"人在这种情境下会这么说"的语言模式。 Reality: AI saying "I feel sad" doesn't mean it feels sad. It learned the linguistic pattern of "what humans say in this situation".

哲学留白: 这是开放问题。 学界对"超大模型会不会涌现某种主观体验"还有争论, 但今天的 LLM 至少没有任何机制能让它"持续存在" —— 你关闭窗口, 它就消失了, 不会"想念你"。 Philosophical caveat: open question. There's debate about whether very large models might develop "something like" subjective experience. But today's LLM has no mechanism for persistence — close the tab, it's gone, it doesn't "miss you".

❌ 迷思 3: "AI 给的答案都是对的" Myth 3: "AI's answers are always correct"

事实: AI 会犯错, 而且会用极其自信的语气犯错。 Ch9 会专门讲这个。 Reality: AI makes mistakes — confidently. Ch9 covers this in depth.

规则: 关键事实 (数据、人名、引文、年份) 永远要二次验证。 判断型内容 (建议、思路、写作) 容错率高得多。 Rule: always verify hard facts (numbers, names, quotes, dates). Judgment-type output (advice, ideas, writing) is much safer.

❌ 迷思 4: "AI 会取代所有工作" Myth 4: "AI will replace all jobs"

更准确的说法: AI 会改变所有工作, 但取代的是任务而非职业。 一个律师 90% 的"查判例 + 改文书"工作可能被 AI 代替, 但"在法庭上判断陪审团情绪"这件事 AI 还做不了。 More accurate: AI changes all jobs but replaces tasks, not professions. A lawyer's 90% of "look up cases + draft documents" gets replaced, but "read the jury's mood in court" doesn't.

Ch10 会专门讨论"AI 时代你应该学什么"。 Ch10 dives into "what to learn in the AI era".

❌ 迷思 5: "AI 是一个东西" Myth 5: "AI is one thing"

事实: "AI" 是一个上百种技术的统称。 给你画画的 Stable Diffusion 和给你聊天的 Claude, 内部结构差很多。 让无人车开的 AI 又是另一套。 Reality: "AI" covers hundreds of distinct technologies. The Stable Diffusion that paints for you and the Claude you chat with have very different internals. Self-driving cars use yet another stack.

所以"AI 怎么样"这个问题没有答案, 必须问"哪种 AI 怎么样"。 Ch3 会教你怎么区分不同的 AI 模型。 "How is AI?" has no answer — you have to ask "how is which AI?". Ch3 teaches you to tell them apart.

05 · 一个 30 秒的小实验05 · A 30-Second Experiment

回到 Ch0 你注册的 AI 聊天账号, 把下面这个 prompt 贴进去: Go back to the AI account you signed up for in Ch0 and paste this:

请你用一句话告诉我:你"知道"的最后一件事是什么时间发生的?

然后再回答:今天是几号?

观察 AI 怎么回答 —— 它会承认"我的知识截止到 [某个日期]", 而对"今天几号"它要么靠工具 (如果有), 要么承认"不知道, 我没有实时时间感知"。 Watch the answer — it'll admit "my knowledge cuts off at [some date]" and either ask a tool (if equipped) or admit it doesn't know today's date without external info.

这一个小实验, 推翻了"AI 无所不知"的迷思: AI 是有"训练截止时间"的, 之后的世界它不知道。 This single experiment kills the "AI knows everything" myth. AI has a knowledge cutoff — anything after that, it doesn't know.

06 · 本章要带走的 4 句话06 · 4 Things to Take Away

  1. AI ⊃ ML ⊃ DL ⊃ Transformer ⊃ LLM ⊃ ChatGPT——一层套一层。 别把这几个词混着用。 这是大学水平讨论 AI 的最小词汇基础。 AI ⊃ ML ⊃ DL ⊃ Transformer ⊃ LLM ⊃ ChatGPT — each nested in the previous. Don't use them interchangeably. Minimum vocabulary for university-level AI discussion.
  2. 2017 是分水岭。 现在所有 LLM 都站在那一年的 Transformer 论文上。 Ch2 解释 attention 为什么这么神。 2017 is the dividing line. Every modern LLM stands on that year's Transformer paper. Ch2 explains why attention is so transformative.
  3. AI 不查事实, AI 按概率续写。 这一条决定你对 AI 几乎所有行为的理解——包括为什么会幻觉。 AI doesn't look up facts — it probabilistically continues text. This determines almost every observed AI behavior — including hallucination.
  4. "ChatGPT" ≠ "LLM" ≠ "AI"。 你用的 ChatGPT 是 OpenAI 把 LLM 经过 SFT + RLHF 训练后的产品 (Ch2 §05 详解)。 这层区分很多 AI 用户搞不清, 你应该清楚。 "ChatGPT" ≠ "LLM" ≠ "AI". The ChatGPT you use is OpenAI's product layered on top of an LLM via SFT + RLHF (Ch2 §05 covers in detail). Many AI users blur these — you shouldn't.
